# 修正内容の確認

1. **`src/app/dashboard/page.tsx`**
   - サーバーコンポーネントで `sort` パラメータを受け取る処理を変更し、デフォルトを `release_asc` としました。
   - クエリの実行順序を「フィルタリング → ソート → リミット」の順に修正しました。これにより、「公開日順」を選択した際に他の月のデータに押し出されてリストが空になる不具合が解消されました。
   - 不要になった `reaction_count` によるソート処理を削除しました。

2. **`src/app/dashboard/dashboard-client.tsx`**
   - 並び替えの選択肢から「自分も観るが多い順」を削除し、「公開日順」と「投稿順」の2種類のみに簡略化しました。
   - それぞれのラベルをユーザーの要望に合わせて「公開日順」「投稿順」に変更しました。
   - 月変更時や並び替え変更時に、`router.push` を使って URL に `?month=...&sort=...` を付与し、サーバーサイドとのソート状態の同期を実現しました。
   - `SelectContent` に `bg-cinema-surface/95` と `z-[60]` を追加し、視認性を向上させました。

3. **`src/components/month-filter.tsx`**
   - 月選択のドロップダウンも同様に視認性を高めるために背景と `z-index` を調整しました。
